# CHAVEIRO System
## Regras Obrigatórias

Este documento contém as regras absolutas do projeto.

Toda IA que trabalhar neste repositório deve seguir estas regras sem exceções.

---

# 1. Nunca simplificar

Nunca remover funcionalidades.

Nunca substituir componentes apenas por serem mais simples.

Nunca alterar a arquitetura existente.

Nunca modificar a organização das pastas.

Nunca criar soluções alternativas quando já existir uma estrutura equivalente.

---

# 2. Sempre reutilizar

Sempre reutilizar:

- componentes
- hooks
- services
- providers
- layouts
- design system

Evitar duplicação de código.

---

# 3. Interface

Toda tela nova deve seguir exatamente:

- cores
- espaçamentos
- sombras
- bordas
- tipografia
- animações

do projeto existente.

---

# 4. Banco

Sempre seguir DATABASE_STRUCTURE.md.

Nunca criar tabelas diferentes.

Nunca alterar nomes de colunas.

Nunca alterar relacionamentos.

---

# 5. Arquitetura

Sempre seguir SYSTEM_ARCHITECTURE.md.

Qualquer funcionalidade nova deve respeitar:

Frontend

Backend

Services

Hooks

Contexts

Providers

---

# 6. IA

Toda IA deve:

analisar primeiro

pensar depois

só então modificar o código.

Nunca modificar arquivos sem antes entender a estrutura.

---

# 7. Objetivo

O objetivo é tornar o CHAVEIRO System o maior ERP para chaveiros do Brasil.

Todas as implementações devem aumentar a qualidade do sistema sem alterar sua identidade.
